## 一、项目是什么

把 **hm-dianping(黑马点评)** 魔改为 **多租户 AI 智能客服 SaaS 平台**:
保留点评沉淀的 Redis/中间件基础设施(鉴权、缓存、分布式锁、ID 生成、Stream 等),
业务主体换成「生产级 AI 客服」,用一套**高并发治理体系**(缓存/限流/削峰/降级/熔断/隔离)驯服又慢又贵的 LLM。

- **技术栈**:Spring Boot 2.3.12 / Java 8 / MyBatis-Plus / Redis Stack(缓存+向量库) / RocketMQ / 阿里百炼(Spring AI) / 多租户逻辑隔离。
- **项目灵魂**:高并发治理。AI 是面子,工程能力是里子。

## 三、本地环境(Docker 一键启动)

编排文件:[`deploy/docker-compose.yml`](./deploy/docker-compose.yml)
启动:`cd deploy && docker compose up -d`

| 服务 | 宿主机连接地址 | 备注 |
|---|---|---|
| MySQL | `127.0.0.1:3308` root/`123456` | 库 `ai_customer_service`,建表脚本放 `deploy/mysql/init/` |
| Redis Stack | `127.0.0.1:6381`(可视化 http://localhost:8001) | 自带 RediSearch 向量检索 |
| RocketMQ NameServer | `127.0.0.1:9876` | — |
| RocketMQ Broker | 广播 `127.0.0.1:10911` | `brokerIP1=127.0.0.1`,宿主机 app 可直连 |
| RocketMQ Dashboard | http://localhost:8089 | — |

> ⚠️ 端口刻意避开本机/WSL/其他容器占用:本机已有 3306 MySQL、另一容器占 3307;本机/WSL 已有 6379 Redis。
> ⚠️ broker 服务加了 `user: root`,否则镜像内 uid 3000 写不进 root 属主的 store 数据卷会崩溃重启。

**大模型(百炼 DashScope)**:API Key 在系统环境变量 `API-KEY`(User 级);后端 `spring.ai.dashscope.api-key: ${API-KEY}`,聊天模型 `qwen-plus`。本地起后端建议用 PowerShell(确保进程能读到该环境变量)。

### 2026-06-11 — M1·Phase C+D:多租户 + 鉴权改造 + 租户/知识库 CRUD
- **建表**:`deploy/mysql/init/01-schema.sql` 新增 `tenant` / `sys_user`(全局唯一 username,带 tenant_id)/ `knowledge_base`(带 tenant_id)。
  > 注意:init 目录只在 MySQL 首次初始化空卷时自动执行;卷已存在,本次是手动 `docker exec ... < 01-schema.sql` apply 的。
- **鉴权改造(多租户版)**:新增 `com.hmdp.auth` 包 —— `UserContext`(ThreadLocal 带 `getTenantId()`)、`RefreshTokenInterceptor`(读 token→登录态含 tenantId,续期)、`LoginInterceptor`(401)。`MvcConfig` 改挂新拦截器,公开 `/auth/login`、`/auth/register`;删除点评旧拦截器 `LoginIntercepter`/`RefreshTokenIntercepter`。
- **登录模型**:`AuthServiceImpl` —— `/auth/register` 事务内建租户+首个 ADMIN 账号;`/auth/login` 校验后把 `LoginUser{id,tenantId,username,nickName,role}` 写入 Redis Hash `login:token:{token}`(24h,续期)。复用点评 `PasswordEncoder`(salt@md5)。
- **CRUD**:`TenantController /tenant/current`;`KnowledgeBaseController /kb`(增/列/查/改/删),服务层一律按 `UserContext.getTenantId()` 过滤,跨租户访问查不到。
- **Redis key 规范**:`RedisConstants.tenantKey(tenantId, suffix)` → `t:{tenantId}:{suffix}`。
- **验证(13 项端到端全过)**:注册两租户→登录→`/auth/me` 带出 tenantId/role→各自建 KB→A/B 互相只看到自己的→**B 越权读 A 的 KB 返回「知识库不存在」**→无 token 401;中文 UTF-8 往返正确。
  > 踩坑:用 curl 从 PowerShell 直接发中文 JSON 会被按 GBK 编码导致服务端 `Invalid UTF-8 0xbf`;改用 UTF-8 文件 `--data-binary @file` 即正常,非代码问题。
