## 一、项目是什么

把 **hm-dianping(黑马点评)** 魔改为 **多租户 AI 智能客服 SaaS 平台**:
保留点评沉淀的 Redis/中间件基础设施(鉴权、缓存、分布式锁、ID 生成、Stream 等),
业务主体换成「生产级 AI 客服」,用一套**高并发治理体系**(缓存/限流/削峰/降级/熔断/隔离)驯服又慢又贵的 LLM。

- **技术栈**:Spring Boot 2.3.12 / Java 8 / MyBatis-Plus / Redis Stack(缓存+向量库) / RocketMQ / 阿里百炼(Spring AI) / 多租户逻辑隔离。
- **项目灵魂**:高并发治理。AI 是面子,工程能力是里子。

## 三、本地环境(Docker 一键启动)

编排文件:[`deploy/docker-compose.yml`](./deploy/docker-compose.yml)
启动:`cd deploy && docker compose up -d`

| 服务 | 宿主机连接地址 | 备注 |
|---|---|---|
| MySQL | `127.0.0.1:3308` root/`123456` | 库 `ai_customer_service`,建表脚本放 `deploy/mysql/init/` |
| Redis Stack | `127.0.0.1:6381`(可视化 http://localhost:8001) | 自带 RediSearch 向量检索 |
| RocketMQ NameServer | `127.0.0.1:9876` | — |
| RocketMQ Broker | 广播 `127.0.0.1:10911` | `brokerIP1=127.0.0.1`,宿主机 app 可直连 |
| RocketMQ Dashboard | http://localhost:8089 | — |

> ⚠️ 端口刻意避开本机/WSL/其他容器占用:本机已有 3306 MySQL、另一容器占 3307;本机/WSL 已有 6379 Redis。
> ⚠️ broker 服务加了 `user: root`,否则镜像内 uid 3000 写不进 root 属主的 store 数据卷会崩溃重启。

### 2026-06-10 — M1·Phase A+B:框架升级 + 配置指向新环境
- **决策**:原设计用 Spring AI,但其要求 Spring Boot 3 + JDK 17;本机默认 JDK 已是 21,遂**整体升级** Spring Boot `2.3.12` → `3.4.5`、Java `8` → `21`(为后续 Spring AI 铺路)。

- **代码适配**:`count()` 返回 `Long`(2 处);`BlogServiceImpl` 自注入加 `@Lazy`(Boot 3 默认禁循环依赖)。
- **配置**:`application.yaml` 数据源指向 MySQL `3308`/库 `ai_customer_service`、Redis 前缀迁 `spring.data.redis` 指向 `6381`;`RedissonConfig` 同步 6381。
